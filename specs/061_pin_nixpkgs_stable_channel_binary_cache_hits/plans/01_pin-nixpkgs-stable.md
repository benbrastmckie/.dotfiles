## Overview

Pin the primary `nixpkgs` flake input from `nixos-unstable` to the stable `nixos-26.05`
channel so the main system draws from Hydra's fully-built binary cache, eliminating the
~29 spurious source builds caused by the unstable revision outrunning Hydra. The
`nixpkgs-unstable` input is retained unchanged (it feeds the `pkgs-unstable` overlay and
`nix-ai-tools`). `update.sh` is changed so `nix flake update` becomes opt-in behind a
`--update` flag rather than running unconditionally on every rebuild.

This plan is authored under a hard PLANNING-ONLY constraint: a Lean build is actively
using the shared `/nix/store` and root disk is at 94%. The flake edit itself is zero-cost
(text only) and is safe to perform anytime. The actual `nixos-rebuild switch` /
`home-manager switch` is a separate, explicitly GATED step (Phase 4) that MUST NOT run
until Tasks 63 and 64 reclaim store space and the user's Lean work completes. No build,
evaluation, or store/lock-touching command is part of the executable scope of Phases 1-3.

## Goals & Non-Goals

**Goals**:
- Change `nixpkgs.url` to `github:NixOS/nixpkgs/nixos-26.05` (one line in flake.nix).
- Preserve `nixpkgs-unstable` and the `nix-ai-tools` -> `nixpkgs-unstable` follow exactly.

  path doing no flake update.

**Non-Goals**:

  update`, `nix eval`, or any store/lock operation during Phases 1-3.
- Touching `flake.lock` directly (it is regenerated by the gated update, not hand-edited).
- Removing or reworking `inputs.niri` (it is live via `inherit niri;`).
- Changing `home-manager`, `sops-nix`, `lean4`, `lectic`, or `utils` inputs.
- Resolving disk-space pressure (owned by Tasks 63/64).

### Phase 2: Make `nix flake update` opt-in in update.sh [NOT STARTED]

**Goal**: Replace the unconditional `nix flake update` with a `--update` flag so routine
rebuilds no longer advance inputs (which is what outran Hydra), while preserving an explicit
update path.

**Tasks**:
- [ ] Add argument parsing near the top of `update.sh` for a `--update` flag (set `UPDATE_FLAKE=1`), preserving the existing `--no-check` handling for `$1`.
- [ ] Replace the unconditional block at `update.sh:19-21`:
      ```bash
      # Update flake inputs
      echo "===> Updating flake inputs..."
      nix flake update
      ```
      with an opt-in guard:
      ```bash
      # Update flake inputs only when explicitly requested (--update)
      if [ "${UPDATE_FLAKE:-0}" -eq 1 ]; then
        echo "===> Updating flake inputs (--update requested)..."
        nix flake update
      else
        echo "===> Skipping flake input update (pass --update to update inputs)"
      fi
      ```
- [ ] Add a comment documenting cadence: update `nixpkgs` (stable) infrequently for security patches via `nix flake update nixpkgs`; update `nixpkgs-unstable` selectively for overlay packages.
- [ ] Ensure `--update` and `--no-check` can be passed together (parse all args, do not assume positional-only).
- [ ] Do NOT execute `update.sh` as part of this phase (it would trigger a build/switch).

**Timing**: 0.75 hours

**Depends on**: none
