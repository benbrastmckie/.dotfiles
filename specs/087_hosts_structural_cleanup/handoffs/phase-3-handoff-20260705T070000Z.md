  `error: infinite recursion encountered` — `pkgs` is constructed from
  `config.nixpkgs.hostPlatform`, so this is a circular fixpoint. Fixed by threading `system`
  through `specialArgs` instead (`inherit system;` added to the `iso` block's `specialArgs` in
  `flake.nix`; extracted module signature is `{ pkgs, lib, system, ... }:`).

**What NOT to Try**:
- Do not use `pkgs.system` to set `nixpkgs.hostPlatform` anywhere in this repo — it is circular
  by construction. Use a `system` string passed via `specialArgs` (or the top-level `let system`
  binding, if still in lexical scope) instead.
- Do not attempt to route `iso` through `mkHost` — `mkHost.nix` unconditionally requires
  `hosts/<name>/hardware-configuration.nix`, which an ISO lacks.
- Do not touch task 68's broken zfs-kernel state — the two `boot.zfs.forceImportRoot` warnings
  on `iso`/`usb-installer` are expected and pre-existing.
